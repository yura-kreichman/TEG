# Модуль «Инструктажи»

## Назначение

Владелец создаёт инструкции для персонала; сотрудники и кандидаты читают их по публичной ссылке и подписывают электронно. Система фиксирует факт, длительность и обстоятельства ознакомления. Модуль автономен: **не зависит** от операторов, зон и других модулей; подписант — всегда внешнее лицо, вводящее данные вручную (даже если он оператор тенанта). Доступен во всех пакетах подписки без ограничений (feature flag существует, но включён везде).

## Инструкции

У тенанта несколько инструкций (например «Общая», «Электромобили», «Батут»). Редактор с минимальным форматированием: Заголовок 1, Заголовок 2, жирный, курсив, подчёркнутый, маркированный и нумерованный списки. Картинки **не поддерживаются** (решено окончательно). Хранение контента — структурированный формат (JSON-структура редактора или ограниченное подмножество HTML с санитизацией), выбор обоснован на Шаге 2.

Статусы: `draft → published → archived`. У опубликованной — публичная ссылка вида `/i/{tenant-slug}/{instruction-slug}` и QR-код (модалка «показать QR» + поделиться — см. изменение решения ниже). Архивная и черновик по публичной ссылке недоступны (страница «инструкция недоступна», без деталей).

## Версии

Правка опубликованной инструкции создаёт новую версию (immutable snapshot текста прежней версии сохраняется). Каждая подпись привязана к конкретной версии. В журнале записи с устаревшей версией помечаются. Действие владельца «Запросить повторное ознакомление» помечает выбранные записи как требующие обновления; повторная подпись человека создаёт **новую** запись (старая не перезаписывается).

## Публичная страница чтения

Без авторизации, mobile-first, вёрстка по `03-design-system.md` (светлая тема по умолчанию). Заголовок, автоматическая оценка времени чтения («≈ N минут», из длины текста), текст инструкции.

Замер времени чтения: от открытия до отправки формы, при скрытии вкладки (`visibilitychange`) таймер на паузу. Замер — вспомогательная метрика, защищать от манипуляций не нужно, но время передаётся с клиента вместе с формой и сохраняется.

Настройка владельца на уровне инструкции «Проверка честности» (bool): при включении кнопка «Готово» неактивна, пока страница не проскроллена до конца (порог ~95%).

## Подписание

Внизу страницы: чекбокс с текстом-константой «Я прочёл всю инструкцию. Мне всё ясно и понятно. Согласен соблюдать требования и правила, упомянутые в данной инструкции.», обязательные поля: Фамилия и Имя, Телефон, Дата рождения; канвас подписи пальцем/мышью (очистить/подписать заново; сохраняется как PNG); кнопка «Готово» активна только при заполненных полях, чекбоксе и непустой подписи (+ скролл-условие при включённой проверке).

Автоматически фиксируются: дата-время сервера, IP, user-agent (распарсенные устройство и браузер), время чтения, id версии инструкции. После отправки — экран «Спасибо, ознакомление зафиксировано».

## Журнал ознакомлений

Кабинет владельца: Настройки → Инструктажи. Две вкладки: «Инструкции» (список: название, статус, версия, число ознакомлений, kebab: редактировать/ссылка и QR/архивировать) и «Журнал ознакомлений».

Колонки журнала: дата и время, фамилия и имя, телефон, дата рождения, время чтения, IP, устройство, браузер, версия (пометка «устаревшая», если инструкция ушла вперёд). Изображение подписи в таблице **не показывается** — только в PDF.

Фильтры: по инструкции, по периоду. У записи — значок скачивания PDF.

Удаление записи владельцем (например, сотрудник уволен): с подтверждением; PDF и подпись удаляются, факт удаления фиксируется в журнале правок (кто, когда, чья запись) — след остаётся, содержимое нет. Экспорта в CSV нет.

## PDF

Один документ на запись: страница «ЗАЯВЛЕНИЕ» — «Я, {Фамилия Имя} (дата рождения {дата}), прочёл всю инструкцию. Мне всё ясно и понятно. Согласен соблюдать требования и правила, упомянутые в данной инструкции.», строка «Дата: {дата-время} - IP адрес: {IP}», изображение подписи, подпись «Подпись:»; затем полный текст ТОЙ версии инструкции, которая была подписана (документ самодостаточен).

Генерация на сервере по требованию (по клику в журнале), не хранится файлом.

## Уведомления

При каждом подписании — владельцу по всем включённым каналам (Telegram, email, push) через общий слой каналов: «{Имя} ознакомился с инструкцией «{Название}» за {время чтения}».

## Макеты и вёрстка

**Публичная страница чтения и подписания** (mobile-first):
- Шапка: маленькая иконка-бейдж + строка «{Название точки/тенанта} · Инструктаж» — человек должен видеть, чей документ подписывает. Ниже крупный заголовок инструкции (24px/600) и строка «время чтения ≈ N минут» с иконкой часов.
- Текст инструкции — обычная типографика документа по дизайн-системе (H1 17px/600, H2 15px/600, тело 14.5px, line-height 1.55). Никаких карточек и декора — это документ, не лендинг.
- Блок подписания отделён тонкой линией: чекбокс согласия (крупный, 22px, скругление 7px) с полным текстом константы; поля Фамилия/Имя в одну строку (два поля), Телефон/Дата рождения в одну строку; канвас подписи: высота ~96px, пунктирная рамка 1.5px (сигнал «здесь расписаться»), белый фон, штрих тёмно-синий (#1B2A8F, как чернила), ссылка «Очистить» мелким текстом справа над канвасом — не кнопка, чтобы не конкурировать с «Готово».
- Кнопка «Готово» (54px, primary, на всю ширину): disabled пока не выполнены все условия; при включённой «проверке честности» и недочитанной странице на кнопке текст «Дочитайте до конца».

**Журнал ознакомлений** (кабинет владельца):
- Вкладки «Инструкции» / «Журнал ознакомлений» — сегмент-контрол. Под ним фильтры: селект инструкции + период.
- Мобильная ширина: записи не таблицей, а трёхстрочными карточками-строками: (1) Имя Фамилия 14.5px/500; (2) дата время · название инструкции, серым 12px; (3) «читал N мин · vX» серым 12px. Справа — квадратная кнопка 38px с иконкой PDF (самое частое действие, один тап).
- Сигнальные маркеры warning-цветом в третьей строке: аномально короткое время чтения (порог: меньше 25% от оценки времени чтения) и «vX — устарела» (версия меньше текущей опубликованной). Без маркеров строка полностью нейтральна.
- Тап по записи → bottom sheet деталей: все поля (телефон, дата рождения, IP, устройство, браузер, точное время чтения, версия) + кнопки «Скачать PDF», «Запросить повторное ознакомление», «Удалить» (красным, с подтверждением).
- Десктопная ширина (>= md): та же вкладка рендерится полной таблицей с колонками: дата и время, фамилия и имя, телефон, дата рождения, время чтения, IP, устройство, браузер, версия, PDF. Сигнальные маркеры — те же, в соответствующих ячейках.

# Модуль «Деньги»

## Назначение

Единый журнал движения наличных и остатки касс. Другие модули не ведут свои деньги — пишут операции сюда.

## Кассы

У каждой зоны своя наличная касса (`MoneyOperation.zoneId`). Есть и операции уровня точки в целом, не привязанные к конкретной зоне (`MoneyOperation.pointId`) — ровно одно из `zoneId`/`pointId` заполнено (CHECK-констрейнт в миграции): аванс/премия сотруднику (модуль «Рабочее время»), пополнение баланса абонемента (модуль «Абонементы», см. `docs/spec/04-game-room.md` и `src/lib/abonement.ts` — отдельного файла спеки у модуля «Абонементы» пока нет). Остаток зоны = сумма операций её журнала. Остаток точки в целом = сумма остатков всех её зон + операции уровня точки (`getPointCashBalance`, `src/lib/zone-balance.ts`) — не отдельное поле, пересчитывается из журнала каждый раз.

## Типы операций

Список расширяемый — модули добавляют свои поверх базовых ниже (например, `advance`/`bonus_payout` у «Рабочего времени», `abonement_topup`/`abonement_topup_cashless`/`revenue_abonement` у «Абонементов» — полный актуальный список типов и то, какие из них физически не в наличной кассе, см. `CASH_EXCLUDED_TYPES` в `src/lib/zone-balance.ts`, а не переписывать здесь при каждом новом модуле).

- **`revenue`** (выручка) — создаётся автоматически при сдаче итогов: наличные — в кассу зоны, безнал (`revenue_cashless`) — учётно, без наличного остатка.
- **`expense`** (расход) — вводится при сдаче итогов; из кассы какой зоны списывать — настройка владельца.
- **`collection`** (инкассация) — оператор вводит сумму, переданную владельцу; касса уменьшается. Подтверждение владельцем не требуется — расхождения решаются на месте.
- **`change_fund`** (размен) — владелец фиксирует внесение наличных на точку.

Формат записи: дата-время, тип, сумма, касса (зона или точка), кто провёл, комментарий.

## Отчётность

Журнал — источник отчёта «Бизнес: расходы и прибыль» и текущего остатка «сколько наличных должно быть на точке». Все суммы — `font-variant-numeric: tabular-nums`; нулевой остаток кассы визуально приглушён (третичный цвет текста, см. `docs/design/design-system.md`), а не тем же весом, что ненулевые суммы.

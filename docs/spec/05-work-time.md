МОДУЛЬ "РАБОЧЕЕ ВРЕМЯ"

НАЗНАЧЕНИЕ. Учёт смен операторов, начисление зарплаты, авансы, премии, скользящий баланс к выдаче. Термин "смена" (shift) используется только в этом модуле — интервал работы оператора. В модуле Счётчики кассовое событие называется "сдача итогов".

СМЕНА. Оператор вводит время "пришёл" и "ушёл" вручную через time picker в 24-часовом формате (колёсики часы:минуты). Смена через полночь — штатный случай: если "ушёл" < "пришёл", конец относится к следующему дню (12:00 → 03:00 = 15 часов). Дата смены = дата начала. Перерывы не вычитаются. В форме смены также поля "аванс" и "премия" (по умолчанию 0).

РЕЖИМ УЧЁТА ВРЕМЕНИ. Атрибут оператора, задаёт владелец: manual (ручной) или auto (check-in/check-out). Переключение действует на новые смены, старые не трогает.

Ручной: оператор вводит "пришёл/ушёл" через time picker постфактум (как описано в разделе СМЕНА); на главном экране PWA активна кнопка "Добавить смену".

Авто: на главном экране PWA вместо неё кнопка "Начать смену" — фиксирует started_at НА СЕРВЕРЕ. После нажатия кнопка превращается в "Закончить смену", под ней живой счётчик отработанного времени (чч:мм), вычисляемый от серверного started_at (тикает локально, истина на сервере — переживает перезапуск PWA и смену устройства). Одновременно у оператора может быть открыта только одна смена. "Закончить смену" открывает bottom sheet подтверждения: интервал и длительность + поля аванс и премия (по умолчанию 0) + кнопка "Завершить". Оператор в авто-режиме НЕ может редактировать время; ошибки (забыл нажать) исправляет владелец обычной правкой с журналом. Автозакрытия смен нет. Открытая смена длиннее 16 часов: оператору при входе — плашка "Смена не закрыта, обратись к владельцу", владельцу в табеле — подсветка "требует правки". Гео-проверка не используется. Telegram-сводка в авто-режиме отправляется в момент завершения смены (в ручном — в момент ввода).

СТАВКА. Часовая, индивидуальная у каждого оператора, задаёт владелец. Расчёт поминутный: начислено за смену = минуты × ставка / 60, округление до копеек, отображение с двумя знаками, tabular-nums. Изменение ставки действует с даты изменения, прошлые смены НЕ пересчитываются: хранится история ставок (rate, effective_from).

ПРЕМИЯ. Поощрение сверх ставки, выдаётся наличными сразу в день начисления из общей кассы точки. Вводит оператор в форме смены, владелец может редактировать или добавлять. Комментарий не требуется. Создаёт операцию bonus_payout в журнале модуля Деньги. В "заработано" премия входит как начисление, в "к выдаче" НЕ входит — деньги уже выданы.

АВАНС. Выдача наличных из общей кассы точки в счёт заработанного. Вводит оператор (в форме смены или отдельно), владелец может редактировать или добавлять. Создаёт операцию advance в журнале Деньги. Настройка владельца: overdraft_allowed — можно ли брать аванс сверх доступного баланса (баланс уходит в минус). При выключенном овердрафте аванс сверх баланса блокируется с понятным сообщением. Настраивается в "Настройки → Рабочее время" (тенантная настройка, общая для всех операторов).

ГРАНИЦА БИЗНЕС-ДНЯ. Общетенантное значение (не поле модуля Рабочее время — расшарено с "Кассой за день", см. docs/spec/01-counters.md): досдача аванса/премии к уже отправленной "Кассе за день" (см. "СТЫК С МОДУЛЕМ ДЕНЬГИ") бакетируется этой же границей. Редактируется на экране "Сводки → Касса за день"; в "Настройки → Рабочее время" — только справочная строка с текущим значением и ссылкой туда же.

БАЛАНС. Скользящий, без периодов и обнуления, отдельного события "выплата зарплаты" НЕ существует. К выдаче = Σ(начислено по ставке) − Σ(авансы) + перенос. Заработано за период (информационный показатель для выбранного фильтра дат) = начислено по ставке + премии. Перенос (balance_carryover) — ручная запись владельца: стартовый баланс при подключении тенанта или корректировка; может быть положительным и отрицательным.

УВОЛЬНЕНИЕ. Отдельной механики нет: владелец выдаёт остаток обычным авансом (баланс в ноль) и деактивирует оператора. Деактивированный сохраняется в истории и отчётах.

ВАЛИДАЦИЯ. Пересечение с другой сменой того же оператора — жёсткая блокировка (физически невозможно быть на двух сменах одновременно), создание/сохранение отклоняется с ошибкой. Длительность смены больше 16 часов — мягкое предупреждение, в стиле модуля Счётчики: отправка всё равно возможна.

РОЛИ И ВИДИМОСТЬ. Оператор видит только своё: табель, ставку, премии, авансы, баланс. Чужие данные скрыты полностью. Владелец видит всех. Редактирование смен, премий, авансов — только владелец. Журнал правок как в Счётчиках: кто, когда, было → стало; оператору компактная отметка (иконка-карандаш/tooltip), владельцу полная история.

СТЫК С МОДУЛЕМ ДЕНЬГИ. Новые типы операций журнала: advance (аванс), bonus_payout (премия) — оба уменьшают ОБЩУЮ наличную кассу точки, к зоне не привязаны, помечаются оператором-получателем. Благодаря этому авансы и премии автоматически участвуют в остатке "сколько наличных должно быть на точке".

СВЯЗЬ СО СДАЧЕЙ ИТОГОВ. Модули независимы (работать может один, итоги сдавать другой). Мягкие напоминания: после сдачи итогов — "не забудь отметить уход"; при вводе смены, если по доступным оператору зонам итоги сегодня не сданы — "итоги сегодня не сданы" (не блокирует).

ИНТЕРФЕЙС ОПЕРАТОРА (PWA). Главный экран: кнопка "Добавить смену" (режим "ручной") или "Начать/Закончить смену" + live-счётчик (режим "авто", см. "СМЕНА" выше) — зависит от режима, назначенного оператору владельцем. Вкладка "Время" в bottom nav (рендерится только при включённом модуле). Содержимое: текущий баланс "К выдаче" (главная цифра), итоги за выбранный период (заработано / ставка / премии / авансы), в режиме "ручной" — кнопка "Добавить смену" (форма: time picker пришёл/ушёл, аванс, премия), список своих смен: дата, интервал, длительность, начислено, премия/аванс мелкой строкой с цветовым кодом. Открытая (незакрытая) смена в этом списке не показывается — только после check-out.

ИНТЕРФЕЙС ВЛАДЕЛЬЦА. Список операторов: тап по карточке = карточка оператора (данные: баланс + табель).

Настройки оператора. Kebab "···" (в списке операторов и в карточке) открывает не меню, а экран "Настройки оператора". Всплывающее меню с действиями удаляется. Тап по карточке оператора по-прежнему открывает данные (баланс + табель).

Структура экрана настроек:
- Шапка: крупный аватар с бейджем камеры (тап → маленький sheet: "Загрузить фото" / "Выбрать аватар"), под ним имя с иконкой карандаша (тап → переименование), статус.
- Группа "ДОСТУП": строки "ПИН-код" (значение ••••, тап → смена ПИН) и "Доступ к зонам" (текущее значение, например "Все зоны", тап → выбор зон).
- Группа "РАБОТА": строки "Ставка в час" (текущее значение, тап → изменение; история ставок сохраняется по спеке), "Учёт времени" (сегмент-переключатель Ручной/Авто прямо в строке, без перехода), "Цветовая метка" (кружок текущего цвета, тап → выбор цвета).
- Внизу экрана красным текстом: "Деактивировать оператора" и приглушённое "Удалить" (с подтверждением). Деструктивные действия только здесь.
- Каждая строка показывает текущее значение прямо в списке. Настройки, добавляемые будущими модулями, регистрируются строками в существующие группы (механизм — как вкладки карточек сущностей в 00-architecture.md).
Из карточки оператора редактирование ставки и режима убирается (ставка остаётся справочно в шапке серым текстом).

Карточка оператора (данные): шапка (аватар/фото, имя, статус, зоны, ставка — только для чтения), блок баланса — "К выдаче" главной цифрой, ниже заработано/ставка/премии/авансы за период-фильтр; кнопки "Выплатить" НЕТ. Владелец может добавить аванс или премию вручную из карточки. Табель: строки смен (дата, интервал, длительность, начислено, премия/аванс цветом; иконка-карандаш у отредактированных). Тап по строке — bottom sheet с деталями и редактированием (время, премия, аванс). Внизу табеля — строка переноса, если есть. Цветовая метка оператора — атрибут карточки сотрудника (используется в сводках, табеле, отчётах).

TELEGRAM-СВОДКА. Отправляется по факту ввода смены в канал тенанта (тот же, что сводки сдач итогов) — в ручном режиме это момент ввода смены (см. "СМЕНА"), в авто-режиме это момент завершения (check-out), не начала:
🕐 СМЕНА: {имя} — {дата} ({день недели})
{начало}–{конец} · {длительность} — {начислено} ₽
💵 Аванс: {X} ₽ — 🏆 Премия: {Y} ₽   (строка только если есть)
💰 К выдаче: {баланс} ₽

# Библиотека иконок

Личная коллекция SVG-иконок, используемая в RentOS через `IconPicker`
(`src/components/icon-picker.tsx`) и API `/api/icon-library` — иконка Точки/
Зоны/Актива/Оператора хранится как `iconKey = "<семейство>:<имя файла без .svg>"`.

## Структура

- `fluent/` — цветные иконки (Fluent), в т.ч. градиентные `fluent-color--*.svg`. Рендерятся как есть (`<img>`), цвета не трогаем.
- `material/` — вторая коллекция, залита сплошным `fill="#FFFFFF"`. Рендерится через CSS `mask-image` (не `<img>`), поэтому реальный цвет иконки — не то, что в файле, а `currentColor` из темы (чёрный в светлой, светлый в тёмной) — при добавлении новых файлов сюда цвет самого SVG неважен, важен только силуэт/альфа-канал.
- `avatars/` — аватарки оператора (см. `Operator.iconKey`, отдельный picker без переключателя семейств — `families={["avatars"]}` в `IconPicker`). Рендерится как `fluent` (`<img>`, цвета/стиль как в файле) — если у вас плоские одноцветные аватарки и хотите, чтобы они тоже красились по теме, скажите, переключим на mask как у `material`.

Все три — плоские папки, без вложенных категорий: поиск идёт по имени файла, вложенность только мешала бы.

## Правило именования файлов (обязательно для поиска по ключевым словам)

Имя файла = ключевые слова через дефис, на английском, в нижнем регистре, без цифр-версий и служебных префиксов:

```
battery-charging.svg
car-front.svg
credit-card.svg
map-pin.svg
```

Чем точнее и типичнее слово (как в обычном поиске), тем легче потом найти иконку через `grep`/поиск по файлам — никакой отдельной базы данных или JSON-манифеста не заводим, файловая система и есть индекс.

Если у иконки несколько смысловых синонимов — перечисляйте через дефис все, а не выбирайте один:

```
car-auto-vehicle.svg
```

Для `avatars/` то же правило, но ключевые слова — по внешнему виду/характеру персонажа (`girl-smiling.svg`, `man-glasses-beard.svg`), не бренд/автор.

## Формат файла

Только `.svg`. Единственное реальное ограничение на имя файла (не на регистр/пробелы/апострофы — это разрешено) — не использовать `/`, `\` и не называть файл `.`/`..` (см. `src/lib/icon-library.ts`, `isIconName`).

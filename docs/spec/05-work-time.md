МОДУЛЬ "РАБОЧЕЕ ВРЕМЯ"

НАЗНАЧЕНИЕ. Учёт смен операторов, начисление зарплаты, авансы, премии, скользящий баланс к выдаче. Термин "смена" (shift) используется только в этом модуле — интервал работы оператора. В модуле Счётчики кассовое событие называется "сдача итогов".

СМЕНА. Оператор вводит время "пришёл" и "ушёл" вручную через time picker в 24-часовом формате (колёсики часы:минуты). Смена через полночь — штатный случай: если "ушёл" < "пришёл", конец относится к следующему дню (12:00 → 03:00 = 15 часов). Дата смены = дата начала. Перерывы не вычитаются. В форме смены также поля "аванс" и "премия" (по умолчанию 0).

СТАВКА. Часовая, индивидуальная у каждого оператора, задаёт владелец. Расчёт поминутный: начислено за смену = минуты × ставка / 60, округление до копеек, отображение с двумя знаками, tabular-nums. Изменение ставки действует с даты изменения, прошлые смены НЕ пересчитываются: хранится история ставок (rate, effective_from).

ПРЕМИЯ. Поощрение сверх ставки, выдаётся наличными сразу в день начисления из общей кассы точки. Вводит оператор в форме смены, владелец может редактировать или добавлять. Комментарий не требуется. Создаёт операцию bonus_payout в журнале модуля Деньги. В "заработано" премия входит как начисление, в "к выдаче" НЕ входит — деньги уже выданы.

АВАНС. Выдача наличных из общей кассы точки в счёт заработанного. Вводит оператор (в форме смены или отдельно), владелец может редактировать или добавлять. Создаёт операцию advance в журнале Деньги. Настройка владельца: overdraft_allowed — можно ли брать аванс сверх доступного баланса (баланс уходит в минус). При выключенном овердрафте аванс сверх баланса блокируется с понятным сообщением.

БАЛАНС. Скользящий, без периодов и обнуления, отдельного события "выплата зарплаты" НЕ существует. К выдаче = Σ(начислено по ставке) − Σ(авансы) + перенос. Заработано за период (информационный показатель для выбранного фильтра дат) = начислено по ставке + премии. Перенос (balance_carryover) — ручная запись владельца: стартовый баланс при подключении тенанта или корректировка; может быть положительным и отрицательным.

УВОЛЬНЕНИЕ. Отдельной механики нет: владелец выдаёт остаток обычным авансом (баланс в ноль) и деактивирует оператора. Деактивированный сохраняется в истории и отчётах.

ВАЛИДАЦИЯ. Пересечение с другой сменой того же оператора — жёсткая блокировка (физически невозможно быть на двух сменах одновременно), создание/сохранение отклоняется с ошибкой. Длительность смены больше 16 часов — мягкое предупреждение, в стиле модуля Счётчики: отправка всё равно возможна.

РОЛИ И ВИДИМОСТЬ. Оператор видит только своё: табель, ставку, премии, авансы, баланс. Чужие данные скрыты полностью. Владелец видит всех. Редактирование смен, премий, авансов — только владелец. Журнал правок как в Счётчиках: кто, когда, было → стало; оператору компактная отметка (иконка-карандаш/tooltip), владельцу полная история.

СТЫК С МОДУЛЕМ ДЕНЬГИ. Новые типы операций журнала: advance (аванс), bonus_payout (премия) — оба уменьшают ОБЩУЮ наличную кассу точки, к зоне не привязаны, помечаются оператором-получателем. Благодаря этому авансы и премии автоматически участвуют в остатке "сколько наличных должно быть на точке".

СВЯЗЬ СО СДАЧЕЙ ИТОГОВ. Модули независимы (работать может один, итоги сдавать другой). Мягкие напоминания: после сдачи итогов — "не забудь отметить уход"; при вводе смены, если по доступным оператору зонам итоги сегодня не сданы — "итоги сегодня не сданы" (не блокирует).

ИНТЕРФЕЙС ОПЕРАТОРА (PWA). Вкладка "Время" в bottom nav (рендерится только при включённом модуле). Содержимое: текущий баланс "К выдаче" (главная цифра), итоги за выбранный период (заработано / ставка / премии / авансы), кнопка "Добавить смену" (форма: time picker пришёл/ушёл, аванс, премия), список своих смен: дата, интервал, длительность, начислено, премия/аванс мелкой строкой с цветовым кодом.

ИНТЕРФЕЙС ВЛАДЕЛЬЦА. Список операторов: тап по карточке = карточка оператора, kebab "···" = действия (фото, имя, PIN, доступ к зонам, цветовая метка, деактивировать, удалить). Карточка оператора: шапка (аватар/фото, имя, статус, зоны, ставка), блок баланса — "К выдаче" главной цифрой, ниже заработано/ставка/премии/авансы за период-фильтр; кнопки "Выплатить" НЕТ. Владелец может добавить аванс или премию вручную из карточки. Табель: строки смен (дата, интервал, длительность, начислено, премия/аванс цветом; иконка-карандаш у отредактированных). Тап по строке — bottom sheet с деталями и редактированием (время, премия, аванс). Внизу табеля — строка переноса, если есть. Цветовая метка оператора — атрибут карточки сотрудника (используется в сводках, табеле, отчётах).

TELEGRAM-СВОДКА. Отправляется по факту ввода смены в канал тенанта (тот же, что сводки сдач итогов):
🕐 СМЕНА: {имя} — {дата} ({день недели})
{начало}–{конец} · {длительность} — {начислено} ₽
💵 Аванс: {X} ₽ — 🏆 Премия: {Y} ₽   (строка только если есть)
💰 К выдаче: {баланс} ₽

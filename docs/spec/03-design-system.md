# Дизайн-система

## Два интерфейса, разные приоритеты

1. **PWA оператора** — «уличный» интерфейс: телефон, солнце, спешка. Поля ввода и кнопки — минимум 52–56px высотой, высокий контраст, минимум текста, один шаг на экран в мастерах. Для числовых полей обязателен `inputmode="numeric"`. Тема по умолчанию — тёмная. Токены и UI-паттерны (карточки, bottom sheet, kebab-действия, чипы статусов) — те же, что в `docs/design/design-system.md`, единой для всех интерфейсов продукта; тёмные значения токенов ждут отдельного прототипа (см. раздел «Тёмная тема — TODO» там же), акцент — активный пресет тенанта, тот же механизм, что в кабинете владельца.
2. **Кабинет владельца** — «рабочий стол»: плотные таблицы, журналы, графики, фильтры по датам. Тема по умолчанию — светлая. Отклонения (разница ≠ 0, предупреждения) — единственные цветовые акценты в таблицах. Плотность здесь — точечное исключение из общего визуального языка (см. ниже): денежные журналы/отчёты остаются компактными табличными (tabular-nums, умеренная высота строки, тонкий бордюр вместо тени), а карточные экраны (точки/зоны/активы/дашборд) получают полный новый визуальный язык.

## Токены

Все цвета, радиусы, отступы — только через CSS-переменные (design tokens).

Три измерения темизации:
- светлая/тёмная (выбор пользователя);
- акцентная схема (настройка тенанта, 5–6 пресетов: зелёная — по умолчанию, синяя, оранжевая, фиолетовая и др.; произвольный выбор цвета не давать).

Каждый пресет проверен на контраст WCAG AA в обеих темах.

Семантические цвета (success/warning/danger) не зависят от акцентной схемы.

## Типографика

См. подраздел «Типографика» в «Визуальном языке» ниже — единственный источник истины по шрифту и масштабу размеров (Onest, размеры 26–32/18/15–16/13px). Здесь фиксируется только то, что не специфично для визуального языка: числовые данные (показания, суммы, счётчики) — всегда `font-variant-numeric: tabular-nums`, чтобы цифры выстраивались в столбцы. Никаких декоративных шрифтов.

## Иконки и изображения

Интерфейсная обвязка (навигация, действия, статусы) — линейные иконки одного набора (Lucide).

Зоны и активы — цветные изображения: встроенная библиотека под типовые аттракционы + загрузка владельцем своего фото.

Цветовая метка актива — обязательный элемент карточки и полей ввода.

## Навигация

**PWA оператора:** нижняя панель вкладок (bottom nav), собирается динамически из модулей, включённых у тенанта; максимум 5 пунктов, остальное — во вкладке «Ещё». Выключенный модуль не рендерится (не показывать заблокированным). Внутри мастера сдачи итогов нижняя навигация скрыта — только Назад/Далее мастера.

**Кабинет владельца:** сайдбар на десктопе, bottom nav на мобильном.

**НИЖНИЙ БАР — «СТЕКЛО».** Фон: самый светлый тон акцентной схемы поверх белого, итоговая непрозрачность 70–75%, backdrop-filter: blur(18px) saturate(170%). Верхняя граница: 0.5px, акцент с прозрачностью ~14%. Активный пункт — акцентный цвет (иконка + подпись weight 600), неактивные — нейтральный с примесью акцента (не чистый серый). Тёмная тема: та же схема, но базовая поверхность тёмная (surface-2, непрозрачность ~75%) с тонировкой акцентом; не использовать светлое стекло на тёмном фоне. Fallback: `@supports not (backdrop-filter: blur(1px))` → тот же цвет с непрозрачностью 96%, без blur (старые WebView и слабые GPU). Контент страницы прокручивается ПОД баром: нижний padding скролл-контейнеров = высота бара + `env(safe-area-inset-bottom)`. Токены `--nav-glass-bg` и `--nav-glass-border` вычисляются из акцентной схемы автоматически (`color-mix` из `--accent`), а не задаются вручную в каждом пресете — новая акцентная схема должна получать стеклянные цвета без дополнительного кода. Badge-точка на пункте бара: семантический danger-цвет, с тонкой обводкой цветом фона бара. Применяется к обоим барам: кабинет владельца и PWA оператора.

## Визуальный язык (референс: Airbnb)

Тон текстов (сохраняется из прежнего раздела «Стиль», не связан с визуальным языком напрямую): глагол первым («Сдать итоги», «Добавить актив»), без «пожалуйста», без восклицательных знаков в системных сообщениях. Ошибки — что случилось и что делать, одним предложением. Дружелюбный, но не «детский»: пользователи — взрослые, считающие деньги.

**Общее ощущение.** Мягко, дорого, дружелюбно. Светлые тёплые поверхности, контент — герой экрана, хром интерфейса почти незаметен. Глубина и качество передаются тенями, скруглениями и анимацией, а не цветом.

**Поверхности и глубина.**
- Фон страниц: тёплый off-white `#FBFBF9` (токен `surface-0`), не чисто-белый.
- Карточки: белые, `border-radius` 16px (крупные блоки 20–24px), почти без бордюров.
- Глубина — слоистыми мягкими тенями, не рамками:
  - `card-rest`: `0 1px 2px rgba(0,0,0,.06), 0 4px 12px rgba(0,0,0,.06)`
  - `card-hover`: `0 2px 4px rgba(0,0,0,.08), 0 8px 24px rgba(0,0,0,.10)`
  - `floating`: `0 8px 28px rgba(0,0,0,.16)`
- Тонкий бордюр `rgba(0,0,0,.06)` только там, где тень не читается (таблицы, инпуты).

**Контролы.**
- Кнопки: pill или radius 12px, высота 48–52px; primary — насыщенный акцент, текст weight 500–600; на нажатие `scale(0.96)` за 100ms.
- Инпуты: высота 52–56px, radius 12px, бордюр `rgba(0,0,0,.12)`; при фокусе бордюр темнеет до `rgba(0,0,0,.8)` БЕЗ цветного glow-кольца.
- Сегменты-переключатели вместо радиокнопок, крупные чекбоксы/свитчи.

**Формы и действия (все интерфейсы).** Подробности и токены — `docs/design/design-system.md` (§3 «Паттерны UI»). Формы создания новой сущности (точка, зона, актив, тариф, оператор, устройство и т.п.) открываются в bottom sheet, не инлайн-формой в потоке страницы. Действия над существующей сущностью (переименовать, удалить, сменить ПИН, деактивировать и т.п.) — только за кнопкой «···» (kebab) на карточке/строке, открывающей bottom sheet со списком действий; деструктивные пункты — в конце списка, с отдельным подтверждением перед необратимым удалением; текстовых ссылок-действий прямо на карточке быть не должно. Кнопка добавления — pill «+ Добавить» в шапке секции либо пунктирная строка-приглашение внутри карточки; плавающая FAB-кнопка не используется ни в одном интерфейсе.

**Типографика.** Шрифт Onest (Google Fonts, кириллица). Заголовок экрана 26–32px/600, секции 18px/600, тело 15–16px/400, подписи 13px `rgba(0,0,0,.55)`. Межстрочный 1.4–1.5. Числовые данные — `font-variant-numeric: tabular-nums`. Иерархия размером и цветом, не тотальной жирностью.

**Анимация** (framer-motion, spring-физика — не линейные CSS-ease).
- Пружины: `type: "spring"`, `stiffness` 300–400, `damping` 28–32.
- Нажатия: `scale` 0.96, мгновенный отклик. Hover карточек: подъём тени + `translateY(-2px)`.
- Модальные на мобильном — bottom sheet: выезд пружиной, ручка-полоска сверху, закрытие свайпом вниз. Шаги мастеров скользят горизонтально.
- Появление списков: stagger +30ms на элемент, fade + `translateY(8px)`.
- Загрузка: скелетоны с мягким мерцанием, не полноэкранные спиннеры.
- Длительности 150–300ms; анимация никогда не блокирует действие пользователя.

**Границы применения.** Полный набор — кабинет владельца, лендинг, онбординг. PWA оператора берёт скругления, spring-нажатия и bottom sheets, но сохраняет уличный контраст: бордюры инпутов не светлее `rgba(0,0,0,.25)`, тени — не единственный разделитель блоков, размер цифр приоритетнее воздуха. Тёмная тема: те же принципы, вместо теней — осветление поверхности на ступень. Акцентные пресеты тенанта сохраняются (зелёный по умолчанию); коралловый добавить шестым пресетом.

### Наблюдения из референсов

1. **Карточка объекта** — фото в верхней части с крупным радиусом (16–20px), поверх фото плавающие чипы (статус-бейдж слева сверху, «избранное»/сердце справа сверху) без собственного фона-плашки, только полупрозрачная подложка ради читаемости.
2. **Заголовок и метаданные — под фото, не поверх**: название, короткая метастрока (даты/хозяин), цена + рейтинг в одну строку — без обрамляющих рамок вокруг текста.
3. **Горизонтальные скролл-ленты карточек** с заголовком секции и шевроном «›» («Popular homes in Philadelphia ›») вместо сетки с пагинацией — компактный способ показать много карточек без перегрузки экрана.
4. **Bottom sheet как основной паттерн** для сложных форм (место/даты/гости, фильтры): ручка-полоска сверху, крупный заголовок, крупные pill-сегменты (Dates/Months/Flexible), закреплённая нижняя панель — второстепенное действие текстом слева (Reset/Clear all), основная чёрная pill-кнопка справа (Next/Show N places).
5. **Чипы быстрых пресетов** (иконка + подпись: Free cancellation, Self check-in, Washer) вместо чекбоксов/дропдаунов для часто используемых фильтров.
6. **Числовые данные визуально выделены**: диапазон цен — гистограмма с двухсторонним слайдером, а не только текстовый инпут; рейтинг — звезда + число инлайн с ценой.
7. **Контекстная всплывающая плашка** («Prices include all fees») поверх контента — короткая, не блокирует взаимодействие, не требует закрытия явным действием.
8. **Нижняя навигация**: 5 вкладок, активная залита акцентным цветом (иконка + подпись), неактивные — нейтральный серый; непрочитанное — маленькая точка-индикатор без числа.
9. **Статусные бейджи на карточках** — короткие пилюли с нейтральным полупрозрачным фоном (Pending, Guest favorite, Original), лёгкий вес текста, не яркие цветные плашки.
10. **Действие «второго уровня»** (View listing) — широкая нейтральная (не акцентная) pill-кнопка внутри карточки; акцентный цвет зарезервирован только для по-настоящему первичных действий (активная вкладка навигации, primary CTA вроде Next/Search).

## Стек

Tailwind + shadcn/ui поверх Next.js. Компоненты shadcn темизируются через те же токены. Никаких инлайновых hex-цветов в компонентах.
